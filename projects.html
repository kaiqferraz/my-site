Automação WEB com Cypress

O projeto automation-exercise foi desenvolvido com foco na automação de testes web, aplicando boas práticas de mercado e utilizando Design Patterns para garantir uma estrutura escalável, organizada e de fácil manutenção. Implementado em JavaScript, o projeto faz uso do Cypress em conjunto com o Cypress Cucumber, permitindo a escrita de cenários de teste em Gherkin dentro da abordagem de BDD (Behavior Driven Development). Além disso, foram utilizados recursos do DevTools para o mapeamento e inspeção de elementos, assegurando maior precisão e confiabilidade nos testes. Essa combinação de tecnologias e práticas resulta em uma automação clara, padronizada e preparada para evolução contínua.

Cenários de Teste Implementados 🔍
  • Login: Testa o login com credenciais válidas.
  • Busca de Produtos: Realiza a busca de produtos e valida os resultados.
  • Adicionar Produto ao Carrinho: Adiciona produtos ao carrinho e valida a inclusão.
  • Validar Produtos no Carrinho: Valida os nomes e preços dos produtos no carrinho.

Teste de Carga/Performance com K6

Desenvolvi um projeto de teste de performance para APIs utilizando K6 e JavaScript, seguindo boas práticas de código e estrutura modular. O projeto inclui cenários de carga ramping, validação de respostas, thresholds estratégicos e relatórios automáticos, garantindo insights claros sobre o comportamento do sistema sob diferentes cargas e mantendo o código escalável e manutenível.

Cenários de Teste Implementados 🔍
  • Simulação gradual: De até 1.500 usuários virtuais (cenários ramping).
  • Checks inteligentes: Para validação de status e respostas das APIs.
  • Monitoramento estratégico de métricas:(tempo médio, percentil 95 e taxa de erro).
  • Relatórios: HTML automáticos para análise rápida.
  • Código: Escalável e de fácil manutenção para novos cenários.

Automação Mobile com Robot framework + appium

Este projeto é uma automação mobile estruturada utilizando Robot Framework integrado ao Appium. Ele segue boas práticas de organização de código, separando claramente recursos, dados, páginas e testes, o que facilita manutenção, escalabilidade e reutilização de keywords.

O projeto inclui:🔍
  • Base e configuração: base.resource e appium-config.json para inicialização do Appium Server.
  • Recursos e locators: Localizadores de elementos organizados em arquivos YAML por página.
  • Páginas e keywords:Keywords separadas por funcionalidades.
  • Dados de teste: Armazenados separadamente, garantindo fácil manutenção de credenciais, endereços e informações de pagamento.
  • Execução de testes: Scripts principais no diretório test/, com logs detalhados em HTML e XML.
